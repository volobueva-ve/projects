# Проект: 
Разработка системы краткосрочного прогнозирования спроса для розничной сети (Retail Demand Forecasting)
## Бизнес-задача: 
Оптимизация управления товарными запасами сети из 10 магазинов для 50 категорий товаров (SKU). 
## Цель:
спрогнозировать ежедневные продажи на 3 месяца вперед для снижения затрат на логистику и предотвращения дефицита товаров (out-of-stock).
## Данные и EDA: 
Исторические данные о продажах за 5 лет (913 тыс. транзакций). Провела разведочный анализ данных (EDA): выявила годовую сезонность, недельные паттерны (пики в выходные) и восходящие тренды продаж.
## Инженерия признаков (Feature Engineering):
Сгенерировала календарные фичи (день, недели, месяц, год).Создала лаговые признаки (Sales Lags) за 1, 7, 14, 30 предыдущих дней.Рассчитала скользящие средние (Rolling Mean) за предыдущие 7 и 28 дней и экспоненциально сглаженные средние для улавливания краткосрочных трендов. Далее удалила занчения у которых были пропущенные значения признаков, чтобы очистить датасет.
## Валидация: 
Реализовала валидацию со сдвигом по времени (Time Series Split) во избежание утечки данных из будущего (data leakage).
## Моделирование и Стек: 
Python, LightGBM, Optuna (для тюнинга гиперпараметров), Pandas, Scikit-learn. Настроила пайплайн автоматического подбора гиперпараметров в Optuna, используя SMAPE в качестве целевой метрики оптимизации, что позволило напрямую максимизировать качество финальных прогнозов на кросс-валидации
## Результаты: 
Достигла значения метрики качества SMAPE = 14.6 ( Symmetric Mean Absolute Percentage Error). Использовалась именно эта метрика качества, так как она масштабируема, то есть учитывает разные магазины и разные товары, и симметрична, то есть защищена от деления на 0. Разработанный подход позволяет точнее планировать закупки и снижает риски кассовых разрывов или избытка товаров на складах.
